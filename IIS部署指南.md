# 值班签到系统 IIS 部署指南

## 第一步：安装IIS

### 1.1 打开Windows功能
1. 按 `Win + R`，输入 `optionalfeatures`，回车
2. 或：控制面板 → 程序 → 启用或关闭Windows功能

### 1.2 勾选IIS组件
找到 **Internet Information Services**，展开并勾选以下选项：

```
☑ Internet Information Services
  ☑ Web管理工具
    ☑ IIS管理控制台
  ☑ 万维网服务
    ☑ 应用程序开发功能
      ☑ .NET Extensibility 3.5
      ☑ .NET Extensibility 4.8
      ☑ ASP
      ☑ ASP.NET 3.5
      ☑ ASP.NET 4.8
      ☑ ISAPI扩展
      ☑ ISAPI筛选器
    ☑ 常见HTTP功能
      ☑ 默认文档
      ☑ 目录浏览
      ☑ HTTP错误
      ☑ 静态内容
    ☑ 性能和功能
      ☑ 动态内容压缩
      ☑ 静态内容压缩
    ☑ 安全性
      ☑ 请求筛选
      ☑ Windows身份验证（可选）
```

点击**确定**，等待安装完成。

---

## 第二步：配置网站

### 2.1 打开IIS管理器
1. 按 `Win + R`，输入 `inetmgr`，回车
2. 或：控制面板 → 管理工具 → Internet Information Services (IIS)管理器

### 2.2 创建网站
1. 在左侧连接面板，展开服务器名称
2. 右键**网站** → **添加网站**

### 2.3 填写网站信息
| 配置项 | 填写内容 |
|--------|----------|
| 网站名称 | `值班签到系统` |
| 物理路径 | `C:\Users\GDNN\Desktop\Work\工程建设` |
| 端口 | `80`（如被占用，可改为8080）|
| 主机名 | 留空 |

点击**确定**

---

## 第三步：配置权限

### 3.1 设置文件夹权限
1. 打开文件夹：`C:\Users\GDNN\Desktop\Work\工程建设`
2. 右键 → 属性 → 安全选项卡
3. 点击**编辑** → **添加**
4. 输入 `IIS_IUSRS`，点击**检查名称** → **确定**
5. 勾选**读取**和**执行**权限
6. 确定保存

### 3.2 添加Everyone权限（如需要）
1. 重复上述步骤
2. 输入 `Everyone`，点击**检查名称** → **确定**
3. 勾选**读取**权限

---

## 第四步：配置默认文档

1. 在IIS管理器中，点击刚创建的**值班签到系统**网站
2. 双击**默认文档**图标
3. 右侧点击**添加**
4. 输入 `index.html`，确定
5. 将 `index.html` 上移到最顶部

---

## 第五步：测试访问

### 5.1 本地测试
在浏览器输入：
```
http://localhost
```
或
```
http://127.0.0.1
```

### 5.2 查看本机IP地址
打开CMD，运行：
```cmd
ipconfig
```
找到 **IPv4地址**，如：`192.168.1.100`

### 5.3 内网访问
其他同事通过浏览器访问：
```
http://192.168.1.100
```
（将IP替换为你实际的IP地址）

---

## 第六步：防火墙设置（如无法访问）

### 6.1 允许IIS通过防火墙
1. 控制面板 → Windows Defender 防火墙
2. 点击**允许应用通过Windows防火墙**
3. 点击**更改设置**
4. 找到并勾选 **World Wide Web服务 (HTTP)**
5. 确定保存

### 6.2 或添加端口规则
1. 高级设置 → 入站规则 → 新建规则
2. 选择**端口** → 下一步
3. 选择**TCP**，输入端口 `80`（或你设置的端口）
4. 选择**允许连接**
5. 配置文件全选
6. 名称：`IIS值班签到系统`
7. 完成

---

## 常见问题

### Q1: 端口80被占用
**解决**：在添加网站时，将端口改为 `8080`，访问时用 `http://IP:8080`

### Q2: 403禁止访问
**解决**：检查文件夹权限，确保IIS_IUSRS有读取权限

### Q3: 404找不到页面
**解决**：确认默认文档中有 `index.html`，且文件存在于目录中

### Q4: 只能本机访问，其他电脑无法访问
**解决**：
1. 关闭Windows防火墙测试
2. 检查是否在同一网段
3. 尝试用IP地址访问而非计算机名

- [ ] 网站已创建并指向正确路径
- [ ] 文件夹权限已设置（IIS_IUSRS读取权限）
- [ ] 默认文档包含index.html
- [ ] 防火墙允许HTTP（端口80）
- [ ] 本机可以访问 http://localhost
- [ ] 其他电脑可以访问 http://你的IP

## 备用：使用其他端口

如果80端口被占用，使用8080端口：

1. 在IIS中编辑网站绑定，将端口改为8080
2. 防火墙添加8080端口规则
3. 访问地址：`http://IP地址:8080`
