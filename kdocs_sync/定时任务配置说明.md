# 云文档同步定时任务配置说明

每天晚上 20:00 自动执行签到数据同步到云文档。

## 方法一：使用 Windows 任务计划程序（推荐）

### 步骤 1：创建批处理文件

已创建 `定时同步任务.bat`，用于执行任务。

### 步骤 2：打开任务计划程序

1. 按 `Win + R`，输入 `taskschd.msc`，回车
2. 或者：开始菜单 → Windows 管理工具 → 任务计划程序

### 步骤 3：创建基本任务

1. 点击右侧【创建基本任务】
2. 名称：`签到数据云同步`
3. 描述：`每天晚上8点自动同步签到数据到金山云文档`
4. 点击【下一步】

### 步骤 4：设置触发器

1. 选择【每天】
2. 点击【下一步】
3. 设置开始时间：`20:00:00`
4. 设置每隔：`1` 天
5. 点击【下一步】

### 步骤 5：设置操作

1. 选择【启动程序】
2. 点击【下一步】
3. 程序或脚本：浏览选择 `定时同步任务.bat`
4. 起始于（可选）：填写 `c:\Users\GDNN\Desktop\Work\工程建设\kdocs_sync`
5. 点击【下一步】

### 步骤 6：完成设置

1. 勾选【当单击"完成"时，打开此任务属性的对话框】
2. 点击【完成】

### 步骤 7：高级设置（可选）

在属性对话框中：

1. **常规** 选项卡：
   - 勾选【不管用户是否登录都要运行】
   - 勾选【使用最高权限运行】

2. **条件** 选项卡：
   - 取消勾选【只有在计算机使用交流电源时才启动】（如果是笔记本）

3. **设置** 选项卡：
   - 勾选【如果任务失败，按以下频率重新启动】
   - 设置重新启动间隔：`10` 分钟
   - 设置尝试重新启动最多次数：`3` 次

4. 点击【确定】

### 步骤 8：测试任务

1. 在任务计划程序中找到【签到数据云同步】
2. 右键点击 → 【运行】
3. 查看同步结果

## 方法二：使用 Python 定时调度器

### 启动定时任务

```bash
cd kdocs_sync
python sync_scheduler.py
```

此命令会启动一个后台进程，每天 20:00 自动执行同步。

### 参数说明

```bash
# 立即执行一次同步
python sync_scheduler.py --now

# 自定义同步时间（例如 21:30）
python sync_scheduler.py --time 21:30

# 作为后台服务运行
python sync_scheduler.py --daemon
```

### 停止定时任务

按 `Ctrl + C` 停止定时调度器。

## 方法三：创建 Windows 服务（高级）

### 安装 pywin32

```bash
pip install pywin32
```

### 创建服务脚本

已创建 `sync_service.py`，可以使用以下命令安装为 Windows 服务：

```bash
# 安装服务
python sync_service.py install

# 启动服务
python sync_service.py start

# 停止服务
python sync_service.py stop

# 移除服务
python sync_service.py remove
```

## 常见问题

### 1. 任务没有按时执行

- 检查任务计划程序中的【上次运行时间】和【上次运行结果】
- 确保计算机在 20:00 处于开机状态
- 检查任务属性中的【条件】选项卡，取消电源限制

### 2. 同步失败

- 检查网络连接
- 检查 AirScript Webhook URL 和脚本令牌是否有效
- 查看日志文件获取详细错误信息

### 3. 如何修改同步时间

- 打开任务计划程序
- 找到【签到数据云同步】任务
- 右键点击 → 【属性】→ 【触发器】→ 【编辑】
- 修改时间为新的时间

### 4. 如何临时禁用定时任务

- 打开任务计划程序
- 找到【签到数据云同步】任务
- 右键点击 → 【禁用】
