# 南宁公司值班签到系统 - 部署说明

## 系统架构

现在系统采用 **Node.js + 文件存储** 架构：
- **后端服务器**：`server.js` - 提供API服务和静态文件托管
- **前端页面**：`index.html` + `script.js` + `styles.css`
- **数据存储**：`data.json` - 存储所有签到记录
- **模式配置**：`mode.json` - 存储签到模式设置

## 部署步骤

### 1. 安装 Node.js

1. 访问 https://nodejs.org/
2. 下载 LTS 版本（推荐 18.x 或更高）
3. 运行安装程序，按默认设置安装
4. 验证安装：
   ```cmd
   node -v
   npm -v
   ```

### 2. 部署文件

将所有文件复制到服务器目录：
```
工程建设/
├── server.js          # 后端服务器
├── index.html         # 主页面
├── script.js          # 前端脚本
├── styles.css         # 样式文件
├── logo.png           # Logo图片
├── data.json          # 数据文件（自动创建）
└── mode.json          # 模式配置（自动创建）
```

### 3. 启动服务器

#### 方式一：直接启动（测试用）

```cmd
cd "c:\Users\GDNN\Desktop\Work\工程建设"
node server.js
```

服务器启动后：
- 本机访问：http://localhost:3000
- 局域网访问：http://<服务器IP>:3000

#### 方式二：使用 PM2 守护进程（生产环境推荐）

1. 安装 PM2：
   ```cmd
   npm install -g pm2
   ```

2. 启动服务：
   ```cmd
   cd "c:\Users\GDNN\Desktop\Work\工程建设"
   pm2 start server.js --name "签到系统"
   ```

3. 设置开机自启：
   ```cmd
   pm2 startup
   pm2 save
   ```

4. 常用命令：
   ```cmd
   pm2 status              # 查看状态
   pm2 logs 签到系统        # 查看日志
   pm2 restart 签到系统     # 重启服务
   pm2 stop 签到系统        # 停止服务
   ```

### 4. 局域网访问配置

#### 查看服务器IP地址

```cmd
ipconfig
```

找到以太网适配器的 IPv4 地址，如：`192.168.1.100`

#### 其他电脑访问

在浏览器中输入：
```
http://192.168.1.100:3000
```

（将 192.168.1.100 替换为实际的服务器IP）

### 5. 防火墙设置

如果其他电脑无法访问，需要开放 3000 端口：

#### Windows 防火墙

1. 打开"Windows Defender 防火墙"
2. 点击"高级设置"
3. 点击"入站规则" → "新建规则"
4. 选择"端口" → "下一步"
5. 选择"TCP"，输入端口 `3000` → "下一步"
6. 选择"允许连接" → "下一步"
7. 勾选所有配置文件 → "下一步"
8. 输入名称："签到系统端口" → "完成"

## 数据备份

### 自动备份

建议定期备份以下文件：
- `data.json` - 所有签到记录
- `mode.json` - 模式配置

### 备份脚本

创建 `backup.bat`：
```batch
@echo off
set backup_dir=backup\%date:~0,4%%date:~5,2%%date:~8,2%
mkdir %backup_dir%
copy data.json %backup_dir%\
copy mode.json %backup_dir%\
echo 备份完成: %backup_dir%
```

## 常见问题

### 1. 端口被占用

如果 3000 端口被占用，修改 `server.js` 最后一行：
```javascript
const PORT = process.env.PORT || 8080;  // 改为其他端口
```

### 2. 数据文件损坏

如果 `data.json` 损坏，可以：
1. 停止服务器
2. 删除 `data.json`
3. 重启服务器（会自动创建空文件）

### 3. 内网无法访问

1. 检查服务器防火墙设置
2. 确认服务器和其他电脑在同一网段
3. 尝试 ping 服务器IP测试连通性

### 4. 数据同步问题

- 确保所有电脑都通过服务器IP访问（不要用 localhost）
- 服务器需要保持运行状态
- 数据会实时同步到服务器

## 系统特点

1. **数据集中存储**：所有签到数据保存在服务器，多电脑共享
2. **实时同步**：任何电脑签到后，其他电脑刷新即可看到
3. **IP限制**：支持内网开放和限制IP两种模式
4. **数据持久化**：服务器重启后数据不丢失
5. **自动备份**：JSON文件格式，易于备份和迁移
